Introduction

The Trading Toolkit is a web-based platform designed to centralise campaign information, activation strategies, customer touchpoints, and marketing resources. It provides a single source of truth for campaign teams globally.

Accessing the System

Logging in (front-end view)

URL:

http://localhost:5173

Steps:

  1. Open your web browser and navigate to the URL above
  2. Use the usual Trading Toolkit login method (Supabase authentication)
  3. Enter your credentials: Email and Password
  4. Click Log In or Sign In
  5. You will be taken to the home page
üìù
Login issues? If you see a login error, check that you are using the correct credentials. Contact your system administrator if you cannot log in.

Accessing the Admin panel

Once logged in, you can toggle into Admin mode:

  1. Look for the Admin toggle in the top navigation bar (top-right corner)
  2. Click the toggle to switch to Admin mode
  3. You will see additional navigation options appear, including a link to /admin
  4. Click Admin in the navigation to open the Admin panel
Top navigation bar with Admin toggle (top-right corner)

URL when in Admin panel:

http://localhost:5173/admin

User roles and permissions

The Trading Toolkit uses two main access levels:

Role Access Level What You Can Do
Standard User View-only Browse all front-end pages, view campaign information, click through to external resources
Admin Edit access All of the above, plus: add/edit/delete content in the Admin panel, manage campaigns, update resources, modify touchpoints
üí°
Can't see the Admin toggle? You may not have admin permissions. Contact your system administrator to request access.

Front-End Pages (View-Only)

This section explains what each front-end page displays and how content is controlled. You cannot edit content directly on these pages. All edits must be made through the Admin panel (Section 4).

3.1 Home / Overview

What this page shows: The home page displays the current campaign overview, including:

  • Campaign hero image
  • Campaign name and year (e.g., "2026 Global campaign")
  • Scope, channels, and activation dates
  • Quick links to other sections
Home page showing the campaign hero card with "The Magic of Joy Holiday Season"

How content is controlled:

  • Campaign metadata is managed in Admin > Campaign (see Section 4.2)
  • Quick links are hardcoded and cannot be edited through the Admin panel

3.2 Customer Touchpoints

What this page shows: An interactive accordion list of platforms (Web, App, Email, Social) with expandable sections showing:

  • A screenshot of the platform
  • Numbered markers indicating specific touchpoints
  • Descriptions including title, description, tier badges, and copy ideas
Customer Touchpoints page showing activation levels and platform accordions

How content is controlled:

  • Platforms and touchpoints are managed in Admin > Touchpoints (see Section 4.6)
  • Screenshots are stored as URLs in the platforms table
  • Marker positions are stored as JSON data

3.3 Insights

What this page shows: Campaign performance data and market insights including infographic cards with maps, statistics, and charts.

Insights page with infographic cards showing flight statistics

Image guidance

Hero images should be:

  • Format: JPG or PNG
  • Dimensions: 1200px width x 600px height (2:1 aspect ratio)
  • File size: Under 500KB for optimal performance
  • Hosted externally: Upload to your image hosting service and paste the full URL
‚ö†Ô∏è
Common mistake Pasting a local file path (e.g., /Users/name/image.jpg) instead of a URL. Always use https:// URLs.

4.3 Ways of Working

What this section controls: The workflow steps displayed on the Activate page.

How to add a new workflow step:

  1. Go to Admin > Ways of Working
  2. Click Add New Step
  3. Fill in: Title, Description, Sort Order, Campaign ID
  4. Click Save
  5. Refresh the Activate page to see your new step
üí°
Easy reordering Use increments of 10 (e.g., 10, 20, 30) for sort order. This makes it easier to insert new steps in between without renumbering everything.

4.5 Customer Touchpoints

What this section controls: Platforms, touchpoints, and marker positions.

‚ö†Ô∏è
Most complex admin section Take care when editing marker positions, as incorrect values can break the visual layout.

How marker positions work

Each touchpoint has marker position data stored as JSON:

[{"top": "120px", "left": "50%"}]
  • top: Vertical position from the top (e.g., 120px, 15%)
  • left: Horizontal position from the left (e.g., 50%, 200px)
üí°
Positioning tip Use percentage values for horizontal positioning to ensure markers scale correctly on different screen sizes.

Quick Start Checklist

Use this checklist when onboarding a new content editor:

New content editor onboarding steps

  • Receive login credentials (email and password)
  • Log in at http://localhost:5173 using Supabase authentication
  • Browse the front-end pages to familiarise yourself with the toolkit
  • Toggle into Admin mode (top navigation bar)
  • Navigate to /admin
  • Explore each admin tab
  • Read this guide (sections 4.1 through 4.7)

  1. Go to Admin > Resources

  3. Click Edit
  4. Change the Description to something new
  5. Click Save
  6. Refresh the Resources page on the front-end
  7. Confirm that your updated description now appears on the card

Troubleshooting

"I can't see my changes"

Possible causes:

  • You haven't refreshed the front-end page
  • You're viewing a cached version of the page
  • The campaign_id in your content does not match the current campaign

Solutions:

  1. Hard refresh: Hold Shift and click refresh (or Cmd+Shift+R / Ctrl+Shift+R)
  2. Clear cache: Settings > Privacy > Clear browsing data
  3. Check campaign_id: Ensure it matches the current campaign

"The carousel is not showing my new card"

Possible causes:

  • The card is marked as inactive
  • The card's campaign_id does not match the current campaign
  • The image URL is broken

Solutions:

  1. Check the Active toggle is enabled
  2. Verify the campaign_id matches
  3. Test the image URL by pasting it in a browser

"I entered the campaign name instead of campaign_id"

This is very common. Go to Admin > Campaign, copy the exact ID (e.g., campaign-2026), and paste it into the campaign_id field.

"My image is not loading"

Solutions:

  1. Test the URL by pasting it in a browser
  2. Ensure it starts with https://
  3. Re-upload to an external hosting service if broken
  4. Check sharing settings allow public access

"I deleted something by accident"

Unfortunately, deletions are permanent. Contact your system administrator if database backups exist.

üí°
Pro tip: Consider marking items as inactive instead of deleting them.

Glossary of Terms

Term Definition
Admin panel The backend interface (/admin) where authorised users manage content.
Campaign ID A unique identifier for a campaign (e.g., campaign-2026). Must be used in all linked content.
CTA Call to Action - a button or link prompting user action.
JSON A data format used to store structured information (e.g., marker positions).
Marker position The location (top and left coordinates) where a marker appears on a screenshot.
Sort order A number determining display order. Lower numbers appear first.
Supabase The backend database and authentication system used by the Trading Toolkit.
Tier badges Labels indicating membership tiers (Executive, Premium, Standard).
