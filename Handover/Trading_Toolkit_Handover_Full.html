Introduction

What is the Trading Toolkit?

The Trading Toolkit is a web-based platform designed to centralise campaign information, activation strategies, customer touchpoints, and marketing resources. It provides a single source of truth for campaign teams globally.

Accessing the System

Logging in (front-end view)

URL:

http://localhost:5173

Steps:

  1. Open your web browser and navigate to the URL above
  2. Use the usual Trading Toolkit login method (Supabase authentication)
  3. Enter your credentials: Email and Password
  4. Click Log In or Sign In
  5. You will be taken to the home page
üìù
Login issues? If you see a login error, check that you are using the correct credentials. Contact your system administrator if you cannot log in.

Accessing the Admin panel

Once logged in, you can toggle into Admin mode:

  1. Look for the Admin toggle in the top navigation bar (top-right corner, see Home screenshot below)
  2. Click the toggle to switch to Admin mode
  3. You will see additional navigation options appear, including a link to /admin
  4. Click Admin in the navigation to open the Admin panel

URL when in Admin panel:

http://localhost:5173/admin

User roles and permissions

The Trading Toolkit uses two main access levels:

Role Access Level What You Can Do
Standard User View-only Browse all front-end pages, view campaign information, click through to external resources
Admin Edit access All of the above, plus: add/edit/delete content in the Admin panel, manage campaigns, update resources, modify touchpoints
üí°
Can't see the Admin toggle? You may not have admin permissions. Contact your system administrator to request access.

Front-End Pages (View-Only)

This section explains what each front-end page displays and how content is controlled. You cannot edit content directly on these pages. All edits must be made through the Admin panel (Section 4).

3.1 Home / Overview

What this page shows: The home page displays the current campaign overview, including:

  • Campaign hero image
  • Campaign name and year (e.g., "The Magic of Joy")
  • Scope, channels, and activation dates
  • Quick links to other sections
Home page showing the campaign hero card with "The Magic of Joy Holiday Season"

How content is controlled:

  • Campaign metadata is managed in Admin > Campaign (see Section 4.2)
  • Quick links are hardcoded and cannot be edited through the Admin panel

3.2 Customer Touchpoints

What this page shows: An interactive accordion list of platforms (Web, App, Email, Social) with expandable sections showing:

  • A screenshot of the platform
  • Numbered markers indicating specific touchpoints
  • Descriptions including title, description, tier badges, and copy ideas
Customer Touchpoints page showing activation levels and platform accordions

How content is controlled:

  • Platforms and touchpoints are managed in Admin > Touchpoints (see Section 4.6)
  • Screenshots are stored as URLs in the platforms table
  • Marker positions are stored as JSON data

3.3 Insights

What this page shows: Campaign performance data and market insights including infographic cards with maps, statistics, and charts.

Insights page with infographic cards showing flight statistics
‚ö†Ô∏è
Limited editing available Only the title and subtitle are admin-controlled via Admin > Insight Pages. Infographic cards and charts are currently hardcoded.

3.4 Activate (Ways of Working)

What this page shows: A step-by-step workflow for campaign activation with numbered cards.

Activate page showing workflow steps in card layout

How content is controlled:

  • Workflow steps are managed in Admin > Ways of Working (see Section 4.3)
  • You can add, edit, reorder, or delete steps

3.5 Omnichannel

What this page shows:

  • A 360¬∞ activation strategy grid showing channels arranged around a central card
  • A horizontal carousel of "Summer Joy Ideas" cards
Omnichannel page with 360¬∞ strategy grid and Summer Joy Ideas section

How content is controlled:

  • 360¬∞ strategy grid: Admin > Omnichannel (see Section 4.4)
  • Summer Joy Ideas carousel: Admin > Omnichannel Ideas (see Section 4.5)

3.6 Calendar

What this page shows: A timeline view of campaign milestones and key dates.

Campaign Calendar showing timeline with milestones
üìù
Currently view-only No admin interface exists yet for editing calendar events. Contact your system administrator if updates are needed.

3.7 Resources

What this page shows: A grid of downloadable assets and external links including thumbnails, category badges, titles, descriptions, and CTA buttons.

Resources page showing grid of resource cards with category badges

How content is controlled:

  • Resource cards are managed in Admin > Resources (see Section 4.7)
  • Cards link to external URLs (Canopy, Figma, Google Drive, etc.)

The Admin panel contains multiple tabs, each corresponding to a different content area.

How to navigate (Once Logged In):

Troubleshooting

"I can't see my changes"

Solutions: Refresh the page, clear cache, or check if the campaign_id matches.

"My image is not loading"

Solutions: Ensure the URL is correct and starts with https://. Local file paths will not work.

Glossary of Terms

Term Definition
Admin panel The backend interface (/admin) where authorised users manage content.
Campaign ID A unique identifier for a campaign (e.g., campaign-2026). Must be used in all linked content.
CTA Call to Action - a button or link prompting user action.
JSON A data format used to store structured information (e.g., marker positions).
Marker position The location (top and left coordinates) where a marker appears on a screenshot.
Sort order A number determining display order. Lower numbers appear first.
Supabase The backend database and authentication system used by the Trading Toolkit.
Tier badges Labels indicating membership tiers (Executive, Premium, Standard).
